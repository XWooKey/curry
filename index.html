前言

Aparecium来自拉丁词根parere(/paˈre.re/)，指“使能看见”，出自同一词根的英文词有我们熟悉的appear（出现、显露），以及再衍生出的appearance（外表、外观），disappear（消失），apparent（显而易见的），以及transparent（透明的），都和“看见”有关

—— 在《哈利波特与密室中》，赫敏曾试着用显形咒让汤姆·里德尔的日记显现文字。

3. 点击Try it按钮,而不是Cancel按钮
4. 在输入框中输入需要添加的省名称（所有省的正确名称都可以在pg中的t_area_code找到,目前所有省都为两字）

平台省市区数据说明

说明

省市区数据分别在两张表，区域码为一般为行政区划代码，直辖市与港澳台略有不同，具体请查看默认数据。

mysql 中的 sys_area_cde

字段名 作用
area_id 省市区的编码
area_name 省市区的中文名称
parent_id 当前的省市区隶属于谁
level 区域等级china,district,p_city,prov_id
json 区域的形状

pg 中的 t_area_code

字段名 作用
area_id 省市区的编码
area_name 省市区的中文名称
parent_id 当前的省市区隶属于谁
level 区域等级china,district,p_city,prov_id
geom 区域的形状
json 区域的形状

导入新数据时需要注意数据嵌套规则：

x 下拉框 首页全国地图
直辖市、香港特别行政区、澳门特别行政区 省、市、区共三级 e.g. 上海市-上海市-徐汇区 市、区共两级 e.g. 上海市-徐汇区
台湾 省、市、区共三级 e.g. 台湾省-台湾省-台湾省 省共一级 e.g. 台湾省

上述中已经给出所使用的表，请获取到自定义的数据之后修改对应表即可。这里必须保证层级关系以及地理化信息格式正确。例如在本系统中将所有省名都改为2字。（该操作在SQL脚本中完成）

平台省市区数据说明

说明

省市区数据分别在两张表，区域码为一般为行政区划代码，直辖市与港澳台略有不同，具体请查看默认数据。

mysql 中的 sys_area_cde

字段名 作用
area_id 省市区的编码
area_name 省市区的中文名称
parent_id 当前的省市区隶属于谁
level 区域等级china,district,p_city,prov_id
json 区域的形状

pg 中的 t_area_code

字段名 作用
area_id 省市区的编码
area_name 省市区的中文名称
parent_id 当前的省市区隶属于谁
level 区域等级china,district,p_city,prov_id
geom 区域的形状
json 区域的形状

导入新数据时需要注意数据嵌套规则：

x 下拉框 首页全国地图
直辖市、香港特别行政区、澳门特别行政区 省、市、区共三级 e.g. 上海市-上海市-徐汇区 市、区共两级 e.g. 上海市-徐汇区
台湾 省、市、区共三级 e.g. 台湾省-台湾省-台湾省 省共一级 e.g. 台湾省

3. 点击Try it按钮,而不是Cancel按钮

省市区数据

说明

省市区数据分别在两张表，区域码为一般为行政区划代码，直辖市与港澳台略有不同，具体请查看默认数据。

mysql 中的 sys_area_cde

字段名 作用
area_id 省市区的编码
area_name 省市区的中文名称
parent_id 当前的省市区隶属于谁
level 区域等级china,district,p_city,prov_id
json 区域的形状

pg 中的 t_area_code

字段名 作用
area_id 省市区的编码
area_name 省市区的中文名称
parent_id 当前的省市区隶属于谁
level 区域等级china,district,p_city,prov_id
geom 区域的形状
json 区域的形状

导入新数据时需要注意数据嵌套规则：

x 下拉框 首页全国地图
直辖市、香港特别行政区、澳门特别行政区 省、市、区共三级 e.g. 上海市-上海市-徐汇区 市、区共两级 e.g. 上海市-徐汇区
台湾 省、市、区共三级 e.g. 台湾省-台湾省-台湾省 省共一级 e.g. 台湾省
