  • 直接获取: 通过SFTP、SCP等命令直接从服务器上获取

导入省数据

此项目仅支持单省，所以部署时需要将当前省的数据填充到下拉框控件。

进入swagger方法可以参考测试.swagger章节

进入swagger, 通过接口初始化当前地市数据
1. 滚动鼠标找到部门管理模块，并点击
2. 找到/dept/provName/{provName}
3. 点击Try it按钮,而不是Cancel按钮
4. 在输入框中输入需要添加的省名称（所有省的正确名称都可以在pg中的t_area_code找到）
5. 点击Execute
6. 返回code:0表示操作成功,data:23表示生成了23个市

省市区数据

说明

省市区数据分别在两张表，区域码为一般为行政区划代码，直辖市与港澳台略有不同，具体请查看默认数据。

mysql 中的 sys_area_cde

字段名 作用
area_id 省市区的编码
area_name 省市区的中文名称
parent_id 当前的省市区隶属于谁
level 区域等级china,district,p_city,prov_id
json 区域的形状

pg 中的 t_area_code

字段名 作用
area_id 省市区的编码
area_name 省市区的中文名称
parent_id 当前的省市区隶属于谁
level 区域等级china,district,p_city,prov_id
geom 区域的形状
json 区域的形状

导入新数据时需要注意数据嵌套规则：

x 下拉框 首页全国地图
直辖市、香港特别行政区、澳门特别行政区 省、市、区共三级 e.g. 上海市-上海市-徐汇区 市、区共两级 e.g. 上海市-徐汇区
台湾 省、市、区共三级 e.g. 台湾省-台湾省-台湾省 省共一级 e.g. 台湾省

默认数据

默认数据支持全中国，数据来源于网络。
准确数据购买地址(一般人我不告诉他)：https://highcharts.com.cn/shopcart
